# Conversion OS — Adoption Roadmap

The pace that works. Rushing this — especially the team phase — is the #1 way
vaults die.

**Phase 1 — Foundation (day 1, 30–60 min).**
Install plugin → empty folder → `/setup` (import-first: bring your website, an
about doc, your client list — the interview confirms instead of interrogating)
→ report card → first win (a client brief or 7-day plan). Connect the folder to
Obsidian. Done when: you've read your own brief and it knew things you forgot.

**Phase 2 — Operate (weeks 1–2).**
`/daily` every morning; `/daily save` before closing. Migrate the top 2–3
clients properly: `/client new` each, log the next meeting of each, generate
the first weekly report from real ledger lines. Top up the "thin" rows from
the setup report card. Done when: a week's reports went out generated-first,
edited-second.

**Phase 3 — Automate (weeks 2–4).**
`/autopilot` with whatever connectors exist (meeting transcripts first —
highest signal per setup minute). Budgets at standard. Add the monthly
`/audit` to the schedule. Done when: Monday's brief contains things you didn't
type.

**Phase 4 — Team (weeks 4–8, only after solo habits stick).**
Pick sync via `/team` (decision tree). Add the most curious teammate first,
not the most senior. Seats + access manifests + scoping. Their day-one
checklist comes from `/team seat`. Done when: a teammate runs `/client brief`
on their account and stops asking you for context.

**Phase 5 — Compound (quarterly).**
`/client health all` weekly · `/audit` monthly · quarterly: re-baseline goals
at renewals, harvest playbooks from archives, refresh `reviewed:` dates,
prune. The vault should feel *lighter* every quarter, not heavier — that's
what the budgets and audits are for.

Anti-pattern, stated plainly: rolling out to the whole team in week 1.
Top-down mandates fail; a working operator with visible results converts the
room.

## Where the product is headed

Explicitly deferred from the current release, in rough order:

- **Remote vault access (MCP)** for laptop-closed cloud routines — today
  `/autopilot` documents local scheduling honestly; a hosted access path ships
  once the local loop is proven.
- **Write-enforcement hook** — `/audit` validates structure today (advisory);
  a blocking pre-write validator lands once false-positive rates are known.
- **GUI pipeline views** over the client frontmatter (Obsidian Bases) — the
  data already supports it.
- **Multi-brand patterns** for holdcos — one OS per operating company, shared
  library.
- **Semantic retrieval** if/when vaults outgrow grep — the frontmatter + slug
  discipline is embedding-ready by design.

Beta feedback steers this order — [open an issue](../../issues).

# Folder CLAUDE.md templates

One per top-level folder. ≤50 lines each (shorter is better): routing and local
rules only — no manuals, no duplication of the root file. `Clients/CLAUDE.md` is the area
firewall; the per-client contract template lives in
`../client/references/workspace.md`.

## Inbox/CLAUDE.md

```markdown
# Inbox — capture, then route

Anything can land here: pasted notes, exports, screenshots-to-text, half thoughts.
Nothing lives here. /daily triages every item to its routed home and deletes the
original. If an item mentions a client, it moves into that client's workspace —
`Clients/{slug}/inbox/` if it can't be filed precisely yet.
Items older than 7 days are a /audit finding.
```

## Company/CLAUDE.md

```markdown
# Company — the firm's own context

Slow-changing facts about {{FIRM_NAME}} itself. One topic per file; ≤150 lines;
`reviewed:` updated whenever a human confirms contents.

| File | Holds |
|---|---|
| profile.md | identity, principles, how we operate, one-liner |
| offers.md | services, packages, pricing, delivery model |
| icp.md | who we serve: roles, pains, buying triggers, real examples |
| brand.md | voice rules, positioning, enemy, words to avoid |
| strategy.md | goals, current quarter, owners |
| team.md | (agency) roster summary — detail lives in Team/ |
| stack.md | tools where decisions/work actually live |
| market/ | competitor and market notes, one file per subject |
| standards/ | service quality bars — the ONLY Company files readable from client scope besides offers.md |

Client-specific facts never live here — they go in `Clients/{slug}/context/`.
```

## Clients/CLAUDE.md

```markdown
# Clients — workspaces & firewall

One folder per client. Each is self-contained: Claude becomes that client's
strategist using only their folder + `Library/` + `Company/standards/` +
`Company/offers.md`.

Cardinal rules — they override convenience:

1. Entering a client folder? Read its `CLAUDE.md` and `profile.md` first.
2. NEVER read, quote, or compare against any sibling folder under `Clients/`.
   Other clients do not exist while you work here.
3. Client voice comes from the client's own `context/brand.md` — never from
   `Company/brand.md`.
4. Writes from client scope: inside the client folder, a one-line activity entry
   to today's `Daily/` note, and tasks to the assignee's `Team/{person}/tasks.md`.
   Nothing else, anywhere.
5. Lifecycle status lives in `profile.md` frontmatter. The folder moves exactly
   once — to `_archive/` at end of engagement, via /client archive.
6. New client = /client new. Never hand-roll a workspace.

`_archive/` is cold storage: excluded from reads via .claudeignore, kept for
history and pattern-mining.
```

## Projects/CLAUDE.md

```markdown
# Projects — internal initiatives

Non-client work only (client deliverables live in `Clients/{slug}/work/`).
One folder per project, kebab-case. `README.md` always — overview, status, owner,
next steps, deadline. Subfolders (`research/`, `drafts/`, `notes/`) only when the
first file arrives. Frontmatter: `type: project`, `status: active|on-hold|done`.
Done projects: set status, move folder to `Ops/decisions/archive/` if worth keeping.
```

## Ops/CLAUDE.md

```markdown
# Ops — how the firm runs

| What | Where | Convention |
|---|---|---|

| Internal decisions | decisions/ | `YYYY-MM-DD-{topic}.md`: decision, reasoning, owner |

| Book of business | pipeline.md | GENERATED by /client health all — never hand-edit |

Client meetings do NOT go here — they live in `Clients/{slug}/meetings/`.
```

## Daily/CLAUDE.md

```markdown
# Daily — the operating log

`YYYY-MM-DD.md` per day: morning brief at top, session logs appended, open loops
at bottom. Weekly: `YYYY-Www-review.md`. Session logs include topics, decisions,
files changed, open loops — written by /daily save, scannable in 20 seconds.
One-line activity entries from client scope land under `## Activity`.
[agency] This folder is the firm-wide view; personal day logs live in
[agency] `Team/{person}/daily/` when someone wants their own lane.
```

## Team/CLAUDE.md (agency mode)

```markdown
# Team — people and seats

One folder per person, kebab-case. `profile.md` (role, strengths, working style,
contact norms) · `access.md` (frontmatter `clients: [slugs]` — the seat manifest
/team uses to generate that person's machine-local .claudeignore) · `tasks.md`
(personal board) · `daily/` on demand.
Tasks assigned from any scope land in the assignee's `tasks.md`.
Offboarding = /team offboard: archives the folder, reassigns open tasks.
```

## Library/CLAUDE.md

```markdown
# Library — reusable assets

The firm's productizable kit. If it's useful twice, it lives here.

| Folder | Holds |
|---|---|
| playbooks/ | how we deliver each service, step by step, with quality bars |
| templates/ | document skeletons: reports, proposals, briefs, meeting notes |
| prompts/ | reusable prompts worth keeping |
| styles/ | output styles: how to write an email, report, proposal, post |
| swipe/ | external reference material worth imitating |

Hard rule: nothing in Library/ may link to or quote `Clients/` content. Client
lessons enter only via /client harvest — anonymized. This keeps the entire
folder shippable as a product at any moment.
```

# Team — people and seats

One folder per person, kebab-case. `profile.md` (role, strengths, working style,
contact norms) · `access.md` (frontmatter `clients: [slugs]` — the seat manifest
/team uses to generate that person's machine-local .claudeignore) · `tasks.md`
(personal board) · `daily/` on demand.
Tasks assigned from any scope land in the assignee's `tasks.md`.
Offboarding = /team offboard: archives the folder, reassigns open tasks.

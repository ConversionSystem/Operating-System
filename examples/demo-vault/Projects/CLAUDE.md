# Projects — internal initiatives

Non-client work only (client deliverables live in `Clients/{slug}/work/`).
One folder per project, kebab-case. `README.md` always — overview, status, owner,
next steps, deadline. Subfolders (`research/`, `drafts/`, `notes/`) only when the
first file arrives. Frontmatter: `type: project`, `status: active|on-hold|done`.
Done projects: set status, move folder to `Ops/decisions/archive/` if worth keeping.
